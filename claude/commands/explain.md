# Explain

Explain the current project or a specific file/feature concisely.

## Instructions

1. If no argument given: explain the current working directory — what the project does, its structure, and the main entry points.
2. If an argument is given (file path, function name, or concept): explain just that thing.

Format:
- **What it is** — one sentence
- **How it works** — 2–4 sentences max, at the right level for a senior engineer
- **Where to start** — the most important file(s) or entry point(s)

No padding. No "Great question!". Skip anything that's obvious from the code.
